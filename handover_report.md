## 🛠️ 구현 완료된 핵심 기술 사양 및 히스토리

### 1. 실시간 다중 데스크 동기화 및 스캔 자동화
* **다중 탭 동기화**: 브라우저의 양방향 방송망 채널 API인 `BroadcastChannel`(`mice_idcard_sync` 채널)을 활용해 0.1초 내로 모든 탭(스캐너, 대시보드, 등록)의 데이터베이스 및 출력 횟수를 실시간 동기화합니다.
* **스캐너 전역 감지**: USB HID 바코드/QR 스캐너가 텍스트를 고속 타이핑한 후 `Enter`를 치는 행위를 브라우저 전역 `keydown` 훅으로 캡처해 입력창 포커스 없이도 인쇄를 즉시 실행합니다.
* **음향 피드백**: Web Audio API 기반 가상 비프음을 재생하여 성공 시 "청아한 고음", 실패 시 "경고용 저음"으로 스태프들의 청각 피드백을 지원합니다.

### 2. 짤림 없는 1페이지 인쇄 보장
* **인쇄 매체 제어**: CSS `@media print`를 조밀하게 튜닝하여 명찰 라벨 전용 컨테이너 이외의 모든 관리용 헤더/네비게이션 UI를 완벽히 격리하고 숨겼습니다.
* **백지 출력 방지**: 크롬의 비동기 인쇄 상태 갱신으로 인해 백지가 인쇄되는 오류를 인쇄 완료 대화상자 종료 감지(`afterprint` 이벤트) 시에 인쇄 데이터 상태를 리셋하도록 동기화 락을 적용하여 해결했습니다.
* **외국인/글로벌 텍스트 대응**: 성명 및 소속명의 영문/한글 글자 수를 판별하여 폰트 크기를 단계적으로 다운스케일링하며, 이름과 직책을 세로형 단어 단위 자동 줄바꿈(`whiteSpace: 'normal'`, `wordBreak: 'keep-all'`)으로 재구성하여 어떠한 긴 텍스트도 영역 밖으로 깨지거나 잘려 인쇄되지 않도록 보장했습니다.

### 3. 용지 사이즈 실시간 변수 연동
* 명찰 용지의 가로/세로 mm를 설정 패널(`SettingsPanel.tsx`)에서 변경하면, 브라우저의 CSS 변수(`--page-width`, `--page-height`)에 바인딩되어 실시간으로 명찰 디자인 비율 미리보기 및 `@page` 인쇄 엔진 크기에 실시간 적용됩니다.

### 4. 모바일 현장 셀프 즉석 등록 시스템 (`PublicRegister.tsx`)
* **단독 퍼블릭 진입로**: URL 파라미터 `?view=public-register`가 켜져 있으면 관리자 탭을 전면 비활성화하고 스마트폰 전용 셀프 등록 폼만 가득 차게 렌더링합니다.
* **수집 필드 보강**: 이름, 소속, 직책 외에 필수적으로 **연락처(Phone)**, **이메일(Email)**을 수집하며, **개인정보 수집 및 이용 동의 약관 체크** 기능을 포함하여 보안 규정을 준수합니다.
* **모바일 QR 입장권 발급**: 등록이 완료되는 즉시 브라우저가 모바일 바코드/QR 티켓 화면으로 전환되며, 해당 QR을 데스크 요원의 스캐너에 대면 1초 만에 명찰 인쇄가 실행됩니다.

### 5. 다중 스캐너 보안 로그인 게이트 (`LoginGate.tsx`)
* **비밀번호 전용 인증**: ID 구분 없이, 비밀번호만 치면 그에 해당하는 데스크 ID와 권한으로 자동 로그인되는 스마트 게이트를 제공합니다.
  * **Desk-01 ~ 04 자동 매핑**: `gdw2026d1` ~ `gdw2026d4`
  * **마스터 관리자 (전체 권한)**: `gdw2026admin`
* **보안 역할 관리**: 일반 데스크 계정으로 로그인하면 대시보드와 설정 탭이 화면에서 완전히 숨겨져 조작 및 데이터 유출을 막고, 스캐너 탭이 기본 화면으로 낙하합니다.
* **로그아웃 관리**: 상단 우측에 로그아웃 기능 및 로그인 세션 정보 뱃지를 탑재하여 유동적인 보안 세션 관리가 가능합니다.

### 6. 와이드 데이터 테이블 및 발급 일시 가시화
* 데이터 관리 페이지의 테이블 뷰포트를 화면 전체 폭(98%)으로 넓혀 연락처/이메일 등의 상세 컬럼이 겹치지 않게 조절했습니다.
* 참석 상태 열을 **'발급 일시'** 열로 갱신하여, 인쇄가 완료된 대상은 **`MM-DD HH:MM` (예: `07-09 01:40`)**으로 발급 시간과 날짜를 직관적으로 파악할 수 있도록 갱신했습니다.

### 2. 대량 문자 발송용 CSV/엑셀 다운로드 및 개별 티켓 링크 분리
* **대량 문자 엑셀 내보내기**: `SmsSimulator.tsx` 화면 하단에 엑셀 다운로드 버튼을 추가하여, 전체 사전등록자 명단의 수신번호, 이름, 소속, 등록코드와 함께 각각의 **개인 모바일 티켓 링크**를 포함하는 CSV 파일을 UTF-8 BOM 인코딩 형식으로 다운로드할 수 있게 구현했습니다.
* **열 세분화**: 대량 문자 플랫폼(뿌리오, 쿨에스엠에스 등)의 치환(머지) 기능을 용이하게 쓸 수 있도록 **`등록코드`와 `티켓링크`를 별도의 엑셀 열(Column)로 완벽히 분리**했습니다.

### 3. 모바일 QR 티켓 즉시 조회 연동
* `PublicRegister.tsx`에서 URL 쿼리 파라미터(`?view=public-register&code=10001`)로 유저 등록코드(code)가 주어지면, 셀프 등록 폼 작성을 건너뛰고 **해당 사전등록자의 모바일 QR 입장권 화면이 직접 즉시 표시**되도록 연동 로직을 추가했습니다. 이로써 엑셀 다운로드로 발송된 문자 링크 클릭 시 실제 입장권 조회가 즉각 수행됩니다.

### 4. Supabase 연동 인프라 세팅 및 보안 강화
* **SDK 설치**: `@supabase/supabase-js` 라이브러리를 설치 완료했습니다.
* **초기화 파일 작성**: 환경 변수로부터 Supabase 클라이언트를 구동하는 `src/supabaseClient.ts` 파일을 생성했습니다.
* **환경 변수 템플릿**: API 정보를 담을 프로젝트 루트 내 `.env` 파일 구조를 구성했습니다.
* **보안 누출 예방**: 민감한 `.env` 키값이 깃허브에 커밋되어 유출되지 않도록 `.gitignore` 파일에 환경 변수 필터링 설정을 적용했습니다.
